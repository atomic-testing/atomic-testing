# API overview

## Locator

In a nutshell, locators represent a way to find element on a page or within a test element.

<details>
  <summary>Available locators</summary>

| Locator | Description |
| ------- | ----------- |

| [byDataTestId(dataTestId)](./api-docs/core/locators/by-data-test-id.mdx) | Locate by data-testid attribute. |
| [byRole(role)](./api-docs/core/locators/by-role.mdx) | Locate by the value of role attribute. |
| [byCssSelector(selector)](./api-docs/core/locators/by-css-selector.mdx) | Locate by CSS selector. |
| [byCssClass(className)](./api-docs/core/locators/by-css-class.mdx) | Locate by CSS class name. Locate by CSS class is not recommended because it is not reliable and can be changed easily. |
| [byAttribute(name, value)](./api-docs/core/locators/by-attribute.mdx) | Locate element by attribute |
| [byTagName(tagName)](./api-docs/core/locators/by-tag-name.mdx) | Locate by HTML tag name (not recommended). |
| [byValue(value)](./api-docs/core/locators/by-value.mdx) | Locate by the value of value attribute |
| [byInputType(type)](./api-docs/core/locators/by-input-type.mdx) | Locate by input element by its type, such as text, radio, checkbox etc. |
| [byName(name)](./api-docs/core/locators/by-name.mdx) | Locate by the value of name attribute. |
| [byLinkedElement()](./api-docs/core/locators/by-linked-element.mdx) | (Experimental) Locate an element by matching attributes from another element |
| [byChecked(checked)](./api-docs/core/locators/by-checked.mdx) | Locate checkbox which is checked, usually it can be chained with checkbox locator |

</details>
