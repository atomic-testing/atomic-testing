# API overview

This page is the hand-authored entry point into the generated API reference
(the **API Documentation** section in the sidebar below this page, produced
from the TSDoc comments in each package's source). Read this page first.

## How to read the API reference

The generated reference is organized **by package** — one section per
`@atomic-testing/*` package, mirroring the `packages/` directory. Locators and
framework/interactor packages (React, Vue, Angular, Playwright, Storybook) each
get one section; the Material UI driver packages get one section **per MUI
major version** (`component-driver-mui-v6`, `-v7`, `-v9`, and the MUI-X
equivalents), and the Angular Material driver packages get one section **per
Angular major** (`component-driver-angular-material-v20`, `-v21`, `-v22`) —
same idea, different framework.

Driver **behavior** is consistent across those per-major-version sections — the
same driver names and methods exist in each, because they document the same
component-driver API surface. What differs is the underlying library major
version's DOM, which is why each major has its own package and its own
generated section. In practice this means: **pick the section that matches the
major version your project has installed, and skip the rest** rather than
reading every version's pages. If you're not sure which major you're on, see
the [Framework Guide](./framework-guide.mdx#-component-driver-libraries) for
how to choose.

## Locator

In a nutshell, locators represent a way to find element on a page or within a test element.

<details>
  <summary>Available locators</summary>

| Locator                                                                          | Description                                                                                                            |
| -------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| [byDataTestId(dataTestId)](./api/@atomic-testing/core/functions/byDataTestId.md) | Locate by data-testid attribute.                                                                                       |
| [byRole(role)](./api/@atomic-testing/core/functions/byRole.md)                   | Locate by the value of role attribute.                                                                                 |
| [byAriaLabel(value)](./api/@atomic-testing/core/functions/byAriaLabel.md)        | Locate by the verbatim `aria-label` attribute — often composed with `byRole` to tell same-role siblings apart.         |
| [byCssSelector(selector)](./api/@atomic-testing/core/functions/byCssSelector.md) | Locate by CSS selector.                                                                                                |
| [byCssClass(className)](./api/@atomic-testing/core/functions/byCssClass.md)      | Locate by CSS class name. Locate by CSS class is not recommended because it is not reliable and can be changed easily. |
| [byAttribute(name, value)](./api/@atomic-testing/core/functions/byAttribute.md)  | Locate element by attribute                                                                                            |
| [byTagName(tagName)](./api/@atomic-testing/core/functions/byTagName.md)          | Locate by HTML tag name (not recommended).                                                                             |
| [byValue(value)](./api/@atomic-testing/core/functions/byValue.md)                | Locate by the value of value attribute                                                                                 |
| [byInputType(type)](./api/@atomic-testing/core/functions/byInputType.md)         | Locate by input element by its type, such as text, radio, checkbox etc.                                                |
| [byName(name)](./api/@atomic-testing/core/functions/byName.md)                   | Locate by the value of name attribute.                                                                                 |
| [byLinkedElement()](./api/@atomic-testing/core/functions/byLinkedElement.md)     | (Experimental) Locate an element by matching attributes from another element                                           |
| [byChecked(checked)](./api/@atomic-testing/core/functions/byChecked.md)          | Locate checkbox which is checked, usually it can be chained with checkbox locator                                      |

</details>

## Framework & Interactor Packages

These packages are not component-driver libraries — they don't expose a table
of per-component drivers. Instead each implements the `Interactor` (or, for
`core`/`dom-core`, the abstractions the `Interactor` builds on) for one
environment: creating the `TestEngine`, rendering/bootstrapping the component
under test, and wrapping driver calls with the environment's reactivity model
(React's `act()`, Vue's `nextTick()`, Angular's `bootstrapApplication`). See
the [Framework Guide](./framework-guide.mdx) for help choosing between them;
each row below links into its generated API section for the full method list.

| Package                                                                     | Purpose                                                                                                                                                                |
| --------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [@atomic-testing/core](./api/@atomic-testing/core/index.md)                 | Core abstractions every other package builds on: `ComponentDriver`, the `Interactor` interface, `partTypes`, and the locator builders.                                 |
