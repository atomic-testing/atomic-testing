Setup

Prefer to scaffold?

create atomic-testing detects your runner and writes this config — plus a runnable example test — for you. Reach for this page when you're hand-configuring a runner or want to understand what the scaffolder emits.

Configure your test runner​

Atomic Testing renders your component through your framework's real reconciler (React's act(), Vue's nextTick()), so before writing any test you need a runner that gives you a DOM to render into and compiles TSX. The configs below are the same ones used by this repo's own test suites (see package-tests/component-driver-html-test).

  • Jest
  • Vitest
pnpm add -D jest jest-environment-jsdom @swc/jest @swc/core @types/jest
jest.config.cjs
/** @type {import('jest').Config} */
module.exports = {
  testEnvironment: 'jsdom',
  transform: {
    '^.+\\.(t|j)sx?$': ['@swc/jest', { jsc: { transform: { react: { runtime: 'automatic' } } } }],
  },
  testMatch: ['**/*.test.@(ts|tsx|js|jsx)'],
  // ReactInteractor wraps every interaction in React's act(). Without this flag,
  // React 18+ logs "not wrapped in act(...)" warnings even though it is.
  globals: {
    IS_REACT_ACT_ENVIRONMENT: true,
  },
};
CommonJS config

The .cjs extension keeps this module.exports config as CommonJS even when your package.json has "type": "module", sidestepping the rename trap. (This is why the scaffolder emits jest.config.cjs.)

Testing Vue? The IS_REACT_ACT_ENVIRONMENT global and the JSX transform above are React-specific — use the Vue Jest config below instead.

package.json
{
  "scripts": {
    "test": "jest"
  }
}
pnpm add -D vitest jsdom
vitest.config.ts
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    environment: 'jsdom',
    setupFiles: ['./vitest.setup.ts'],
  },
});
vitest.setup.ts
// Same act()-environment flag as the Jest config above — required whenever
// you're testing React components through `@atomic-testing/react-19` (or
// react-18 / react-legacy).
(globalThis as typeof globalThis & { IS_REACT_ACT_ENVIRONMENT?: boolean }).IS_REACT_ACT_ENVIRONMENT = true;

Vue: Jest with export conditions​

Vue 3 + Jest is a verified stack. Install the Jest runner as in the Jest tab above, then use this config — it drops the React IS_REACT_ACT_ENVIRONMENT global (VueInteractor calls Vue's own nextTick()) and adds testEnvironmentOptions.customExportConditions, without which jsdom resolves the wrong Vue build and @testing-library/vue throws at load:

jest.config.cjs
/** @type {import('jest').Config} */
module.exports = {
  testEnvironment: 'jsdom',
  // Resolve Vue's Node export condition, or @testing-library/vue loads a build
  // that throws under jsdom.
  testEnvironmentOptions: {
    customExportConditions: ['node', 'node-addons'],
  },
  transform: {
    '^.+\\.(t|j)sx?$': '@swc/jest',
  },
  testMatch: ['**/*.test.@(ts|tsx|js|jsx)'],
};

This is exactly what create atomic-testing emits for Vue + Jest. For the package list, see the Vue tab in Manual Installation.

Angular: Vitest browser mode​

Angular has no jsdom/Jest path — its createTestEngine bootstraps a standalone component, which needs a real browser. The supported runner is Vitest browser mode driving Chromium through Playwright. This path is experimental: the scaffolder composes it best-effort and prints a warning before writing.

pnpm add -D vitest @vitest/browser-playwright playwright
vitest.config.ts
import { playwright } from '@vitest/browser-playwright';
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    globals: true,
    setupFiles: ['./vitest.setup.ts'],
    browser: {
      enabled: true,
      provider: playwright(),
      instances: [{ browser: 'chromium' }],
    },
  },
});
vitest.setup.ts
// Angular needs the JIT compiler and zone.js bootstrapped before any component renders.
import '@angular/compiler';
import 'zone.js';
Vite Angular plugin required

Browser-mode Vitest still needs a Vite Angular plugin (e.g. @analogjs/vite-plugin-angular) added to plugins to compile your components — the config above is a starting point.
