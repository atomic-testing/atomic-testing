API overview

This page is the hand-authored entry point into the generated API reference (the API Documentation section in the sidebar below this page, produced from the TSDoc comments in each package's source). Read this page first.

How to read the API reference​

The generated reference is organized by package — one section per @atomic-testing/* package, mirroring the packages/ directory. Locators and framework/interactor packages (React, Vue, Angular, Playwright, Storybook) each get one section; the Material UI driver packages get one section per MUI major version (component-driver-mui-v6, -v7, -v9, and the MUI-X equivalents), and the Angular Material driver packages get one section per Angular major (component-driver-angular-material-v20, -v21, -v22) — same idea, different framework.

Driver behavior is consistent across those per-major-version sections — the same driver names and methods exist in each, because they document the same component-driver API surface. What differs is the underlying library major version's DOM, which is why each major has its own package and its own generated section. In practice this means: pick the section that matches the major version your project has installed, and skip the rest rather than reading every version's pages. If you're not sure which major you're on, see the Framework Guide for how to choose.

Locator​

In a nutshell, locators represent a way to find element on a page or within a test element.

Available locators
LocatorDescription
byDataTestId(dataTestId)Locate by data-testid attribute.
byRole(role)Locate by the value of role attribute.
byAriaLabel(value)Locate by the verbatim aria-label attribute — often composed with byRole to tell same-role siblings apart.
byCssSelector(selector)Locate by CSS selector.
byCssClass(className)Locate by CSS class name. Locate by CSS class is not recommended because it is not reliable and can be changed easily.
byAttribute(name, value)Locate element by attribute
byTagName(tagName)Locate by HTML tag name (not recommended).
byValue(value)Locate by the value of value attribute
byInputType(type)Locate by input element by its type, such as text, radio, checkbox etc.
byName(name)Locate by the value of name attribute.
byLinkedElement()(Experimental) Locate an element by matching attributes from another element
byChecked(checked)Locate checkbox which is checked, usually it can be chained with checkbox locator

shadcn/ui components​

component-driver-shadcn-v1 is a pure re-export of component-driver-radix-v1: shadcn/ui components are styled Radix primitives, so their DOM — and therefore their drivers — is identical. It re-exports (rather than subclasses or copies) so that instanceof agrees across both package names, and it tracks component-driver-radix-v1's major version in lockstep. Use whichever package name matches your dependency tree; the driver set is the one documented in the Radix UI section above, unchanged.

Framework & Interactor Packages​

These packages are not component-driver libraries — they don't expose a table of per-component drivers. Instead each implements the Interactor (or, for core/dom-core, the abstractions the Interactor builds on) for one environment: creating the TestEngine, rendering/bootstrapping the component under test, and wrapping driver calls with the environment's reactivity model (React's act(), Vue's nextTick(), Angular's bootstrapApplication). See the Framework Guide for help choosing between them; each row below links into its generated API section for the full method list.

PackagePurpose
@atomic-testing/coreCore abstractions every other package builds on: ComponentDriver, the Interactor interface, partTypes, and the locator builders.
@atomic-testing/dom-coreDOMInteractor — the base DOM implementation extended by the React, Vue, Angular and Storybook interactors.
@atomic-testing/react-coreShared ReactInteractor + createTestEngine implementation (wraps calls in act()); react-18/react-19/react-legacy re-export it pinned to a React peer range.
@atomic-testing/react-18createTestEngine for React 18 — thin re-export of react-core (ADR-003).
@atomic-testing/react-19createTestEngine for React 19 — thin re-export of react-core (ADR-003).
@atomic-testing/react-legacyAdapter for testing React 17 and earlier.
@atomic-testing/vue-3VueInteractor + createTestEngine (calls nextTick()) for Vue 3 apps.
@atomic-testing/angular-coreAngularInteractor + the bootstrapApplication-based async createTestEngine (ADR-013); install the per-major package instead of depending on this directly.
@atomic-testing/angular-20createTestEngine for Angular 20 — thin re-export of angular-core pinned to the Angular 20 peer range.
@atomic-testing/angular-21createTestEngine for Angular 21 — thin re-export of angular-core pinned to the Angular 21 peer range.
@atomic-testing/angular-22createTestEngine for Angular 22 — thin re-export of angular-core pinned to the Angular 22 peer range.
@atomic-testing/playwrightPlaywrightInteractor — browser automation, used for E2E tests across all supported frameworks.
@atomic-testing/storybookStorybookInteractor + play-function helpers for testing Storybook 10 stories.
